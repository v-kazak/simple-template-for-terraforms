# Terraform-шаблон: Виртуальные машины с сетевым балансировщиком в Yandex Cloud

Готовый и простой шаблон Terraform для быстрого развертывания группы виртуальных машин за сетевым балансировщиком нагрузки (Network Load Balancer) в Yandex Cloud.

При инициализации на все машины через `cloud-init` (user-data) автоматически устанавливается и запускается веб-сервер **Nginx**. Балансировщик сразу после создания начинает распределять между ними HTTP-трафик.
Если этот функционал вам не нужен, закомментируйте строку `user-data = ...` в файле `main.tf`.

## Что создает этот шаблон:
- Виртуальную сеть (VPC) и подсеть.
- Заданное количество виртуальных машин (по умолчанию Debian 12, прерываемые, для экономии).
- Сетевой балансировщик (NLB) уровня L4.
- Target-группу, которая автоматически собирает IP-адреса созданных серверов и подключает их к балансировщику.

## Требования
- Установленный Terraform версии >= 0.13.
- Аккаунт Yandex Cloud.
- Полученные `OAuth-токен`, `cloud_id` и `folder_id`.
- Сгенерированный SSH-ключ для подключения.

## Использование

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/v-kazak/simple-template-for-terraforms.git
   cd simple-template-for-terraforms
   ```

2. Создайте файл секретов `secret.auto.tfvars`. 
   *Файлы `secret.*` добавлены в `.gitignore`*
   ```hcl
   token     = "y0_AgA..."                   # Ваш OAuth токен
   cloud_id  = "b1g..."                      # ID вашего облака
   folder_id = "b1g..."                      # ID вашей папки
   ```

3. (Опционально) Ниже приведены настройки по-умолчанию, дли их изменения укажите в файле `terraform.tfvars` необходимые параметры, например `vps_count = 4`
   
   ```hcl
   # --- Настройки виртуальных машин ---
   vps_count     = 2                      # Количество создаваемых серверов
   name          = "web-server"           # Префикс имени сервера (web-server-0, web-server-1, ...)
   zone          = "ru-central1-d"        # Зона доступности
   blns_zone     = "ru-central1"          # Зона для балансировщика
   platform_type = "standard-v3"          # Тип платформы сервера
   cores_count   = 2                      # Количество vCPU
   memory_count  = 2                      # Объем RAM в ГБ
   core_fraction = 20                     # Гарантированная доля vCPU в %
   disc_size     = 15                     # Размер диска в ГБ
   disc_type     = "network-hdd"          # Тип накопителя
   image_id      = "fd8e9t6fpgi13oh7q39f" # ID образа ОС (Debian 12)  
   preemptible   = true                   # Прерываемость ВМ (true = значительно дешевле)

   # --- Настройки доступа и сети ---
   ssh_user            = "debian"                   # Пользователь для SSH-доступа
   ssh_public_key_path = "~/.ssh/id_rsa.pub"        # Путь к вашему публичному SSH-ключу
   nat                 = true                       # Назначать публичный IP-адрес каждой ВМ
   ```

4. Инициализируйте Terraform:
   ```bash
   terraform init
   ```

5. Проверьте план развертывания:
   ```bash
   terraform plan
   ```

6. Примените конфигурацию:
   ```bash
   terraform apply
   ```

## Подключение к серверам

После успешного выполнения `terraform apply`, в выводе вы увидите публичный IP-адрес балансировщика (`load_balancer_public_ip`) и список публичных IP-адресов виртуальных машин (`external_ips`), если `nat = true`.

Для подключения к ВМ по SSH используйте пользователя, указанного в переменной `ssh_user` (по умолчанию `debian`):
```bash
ssh <ssh_user>@<external_ip>
```

Чтобы удалить все созданные ресурсы и перестать за них платить, выполните:
```bash
terraform destroy
```
